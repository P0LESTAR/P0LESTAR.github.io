>『실무자가 말하는 모의해킹』에서 인용한 리스트입니다.  
{: .prompt-info }

---

1. 모의해킹 업무 협의 단계에서 꼭 확인을 해야 할 사항은 무엇일까?

2. 모의해킹을 진행할 때 자동 스캔 도구의 장단점은 무엇인가?
자동 스캔 도구 사용이 필수일 때 유의할 점은 무엇인가?

3. 테스트 환경에서 어떤 자동 스캔 도구를 확인해봤는가?
실무에 활용할 수 있는 최선의 방법은 무엇인지 고민해봤는가?

4. 고객이 테스트 계정을 주지 않는다면 고객에게 어떻게 대응할 것인가?
고객에게 어떤 조언을 줄 수 있을까?

5. 모의해킹 과정에서 작은 장애라도 발생했다면 고객에게 어떤 대응을 할 것인가?

6. 모의해킹 진단 과정에서 서비스 장애를 일으킬 수 있는 항목은 무엇이고, 이 항목들은 어떤 접근 방법으로 진단할 필요가 있는가?

7. 서비스 업종마다 모의해킹 접근 방법과 항목들이 다르다. 분석한 업종 중 2개를 비교해 설명해보자.

8. 진단 대상이 C클라스 10개 이상 주어졌다고 가정해보자. 정보 수집과 대상 시스템 버전의 이력 관리 등을 효율적으로 할 수 있는 방법은 무엇일까?

9. XSS 취약점을 발견했을 때 이를 이용해 서비스에 어떤 영향들을 보여줄 수 있는가? 테스트를 한 환경 내에서 최대한 큰 영향을 고객에게 인식시키려면 어떤 것을 선택해야 할까?

10. XSS 취약점이나 SQL Injection 등 입력 값 검증을 할 때, 어떤 상황들이 시스템에 영향을 줄 수 있는가?

11. 테스트한 환경에서 CSRF를 수행할 수 있는 시나리오들에는 어떤 것이 있는가?
가능성 있는 모든 시나리오를 설명해보자.

12. 파일 업로드 취약점을 이용해 서버를 침투할 때, 이후 2차 또는 3차 공격을 이용해 어떤 것을 획득하겠는가?

13. 웹셸이 방어 시스템에 의해 사전에 탐지될 수 있는지 없는지를 판단하기 위해 테스트 환경을 구성할 경우, 어떤 구성을 할 것인가?

14. 고객이 웹셸 차단 솔루션을 도입하기도 전에 BMT 협조가 이루어졌다면, 어떤 항목들을 체크 할목에 포함시킬 것인가?

15. 시스템 침투 후 사용자 계정의 패스워드를 취득하려 할 때, 가장 효과적인 접근 방법은 무엇일까?

16. 시스템 침투 후 근접 내부 네트워크를 2차적으로 침투해야 한다면 어떤 방법을 이용할 것인가? 백도어가 최대한 탐지되지 않도록 하는 방법에는 무엇이 있는가?

17. 인증 처리 미흡 취약점을 이용해 공격할 수 있는 시나리오를 5가지 말해보자.

18. SQL Injection을 효과적으로 차단할 수 있는 방법은 무엇일까? 상황마다 다르게 적용해야한다면 상황별로 설명해보자.

19. 개인정보 수정 페이지에서 다른 사용자의 개인정보를 획득할 수 있는 취약점이 노출되었다고 하자. 이를 빠르게 가져올 수 있는 방법은 무엇일까?

20. 수행 과정에서 관리자 페이지가 침투됐다.
관리자 페이지가 사전 협의된 대상이 아닐 경우 여기서 중단하는게 좋을까?
아니면 고객사에 어떤 제안을 하는 게 좋을까?

21. IDS/IPS 탐지 시스템을 이용해 Active한 공격을 탐지할 수 있는지 체크하려면, 테스트 환경을 어떻게 구성하면 좋을까?
이때 관련 오픈소스를 자세히 다뤄봤는가?

22. 고객사에서 IDS/IPS 탐지 시스템에 BMT 지원을 요청한다면, 최신 공격 기법 중 어떤 것을 선택할 것인가?

23. 실제 보고서에 제시한 사례별 공격 여부를 확인해볼 경우, 탐지 시스템에 대해 어떤 생각이 들까?

24. 탐지 시스템의 시그니처 룰로 많이 사용하는 Snort Rules 기반으로, 탐지되지 않는 공격에 대해 시그니처 업데이트를 해봤는가?

25. 웹 방화벽 시스템을 최적화해 구성한 뒤 이를 우회할 수 있는 공격 패턴들을 조사해봤는가?

26. 사회 공학적 기법을 이용해 고객사에 어떤 서비스를 제공할 수 있을까?

27. 전 임직원 대상으로 보안 인식 재고 훈련을 위해 제안할 수 있는 방법들에는 무엇이 있을까?

28. 메일 악성 첨부 파일 방식으로 보안 인식 재고를 선택했는데, 스팸 메일 및 악성 코드 방어 시스템에서 차단이 이뤄지고 있다면 어떤 방법이 좋을까?

29. 행자부에서 배포한 가이드에는 웹 취약점의 영향도가 모두 '상'으로 체크돼 있는데, 이것이 맞다고 생각하는가?
틀렸다고 생각하면 왜 그런지 설명해보자. 아울러 위협 및 영향도 분석에 대해서도 논의해보자.

30. 최신 취약점에 대한 공격코드가 배포되고, 자신이 수행했던 고객사에서 해당 서비스를 운영하는 걸 할게 되었다면 어떻게 할 것인가?

31. 개발 단계부터 투입되었을 때 진행 절차는 어떻게 이루어지는가?

32. 소스코드 진단을 수행할 때 자동 소스 도구로만 수행한다면 어떤 문제점이 있을까?(소스코드 진단의 한계.) 이를 보완하기 위한 추가 진단 방법에는 무엇일 있을까?

33. 안드로이드 서비스의 취약점을 진단할 때 적용할 수 있는 진단 항목에 대해 설명해보자.

34. 모바일 보안 시큐어코딩을 적용하려 테스트해본 사례가 있는가?

35. 금융 모바일 앱 진단 체크리스트에 대해 설명해보자.

36. 앱 루팅을 우회하는 기법과 함께 이를 보안할 수 있는 방법을 제시해보자.

37. 사용자들이 많이 사용하는 금융권 및 게임 서비스에 설치돼 있는 보안 솔루션 종류가 무엇이고 어떤 역할을 하는지 얘기해보자.

38. 각 보안 솔루션을 우회할 수 있는 기법들과 함계 이들에 대응할 수 있는 방안을 말해보자.

39. 고객사에서 문서 보안 솔루션 점검을 의뢰했다면, 어떤 항목을 점검해야 하고 어떤 시나리오로 접근해야 할까?

40. 테스트 환경 대상으로 모의해킹을 진단한 뒤 코드 수정과 보안 설정을 직접 해보았는가?

41. 보고서를 작성해야하는 상황인데 양식도 샘플 가이드도 없다면 어떻게 할 것인가?

42. 버그헌팅에 관심이 있다면 국내외 버그헌팅 시장에 대해 얘기해보자. 업무 중에 제로데이 취약점을 찾았다면 어떻게 대응할 건지도 설명해보자.

43. 최근 이슈인 공격 기법에 대해 2가지 이상 설명해보자.
이어 기술적, 관리적 측면에서 어떻게 방어할 수 있는지도 얘기해보자.

44. 컨설팅한 뒤 마스터플랜을 세워 고객에게 제시할 때, 모의해킹 방법론을 실제 업무의 어떤 프로세스에 적용할 것을 제안하고 싶은가?

45. 소스코드를 직접 작성해 모의해킹 진단에 활용한 적이 있는가?
또 어떤 것을 진단할 때 소스 코드 활용도가 높았나?

46. (PM질문) 팀원들이 모든 서비스에 대해 점검 항목을 동일하게 진단하고 있는지 관리하려면 어떤 방법이 좋을까?
